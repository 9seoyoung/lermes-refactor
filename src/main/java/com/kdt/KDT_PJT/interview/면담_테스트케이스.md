## 면담 신청 (강사, 학생만 가능)
로그인후 
`POST` http://onopco2.iptime.org:940/api/interview/apply

"itvPicAuthrt" -> `대표`,`직원`,`강사` 입력 가능.

#### 요청
```json
{
  "itvAplyTtl": "면담 신청 제목",
  "itvAplyCn": "면담 신청 내용",
  "itvPicAuthrt": "직원"
}
```
#### 응답
```json
{
    "itvAplyTtl": "면담 신청 제목",
    "itvAplyCn": "면담 신청 내용",
    "itvPicAuthrt": 3,
    "itvAplcntSn": 312,
    "cohortSn": 100,
    "formUuid": "5648df19a4fc48a696d6c304b342f203",
    "itvAplyDt": "2025-09-23T16:43:03",
    "itvSn": 33
}
```

## 면담 상세 조회

`GET`  /api/interview/read/{itvSn}

itvSn으로 조회 요청 넣음
근데 이게 저거바꿔가면서 다른회사나 기수 사람이 볼수도있으니까 그거를 먼저 처리해줘야함.

1. itvSn으로 상세 조회 요청
2. 슈퍼어드민은 프리패스, 나머지는 조회하려는 글의 회사에 본인이 포함돼있는지, 강사라면 그기수 강사 맞는지 확인
3. 상세조회 리턴

`GET` http://onopco2.iptime.org:940/api/interview/read/38 했을때 예시

응답 
```json
{
  "itvSn": 38,
  "coSn": 15,
  "cohortSn": 100,
  "itvAplyTtl": "면담 신청 제목",
  "itvAplyCn": "면담 신청 내용",
  "itvAplyDt": "2025-09-24T14:29:20",
  "itvPicAutht": 2,
  "viewCnt": 2,
  "itvAplcntNm": "15_100학생"
}
```




## 면담 확정 (대표, 직원, 강사만 가능)
`PUT` /api/interview/confirm/{itvSn}

### 요청
```json
{
  "itvDay": "2025-09-22",
  "itvTime": "09:00",
  "itvPlc": "면담장소",
  "itvPicAns": "면담 신청자에게 할말(요청사항)"
}
```
본인 계급에 해당하면 update 완료, 이미 확정된애 요청 다시 보내면 에러

면담 확정시 면담 예정자 2명의 개인 일정 등록

예시) <br>
`제목` : [면담 일정] 09:00 15_100학생 -> 15_100대표 <br>
`시작시각,종료시각 (동일)` : 2025-09-22 09:00:00
